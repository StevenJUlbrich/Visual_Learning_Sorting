# 11 CI - Pipeline Requirements

Scope: Mandatory CI gates for contract-safe delivery.
Grounded in project stack: Python 3.13 + `uv` + `ruff` + `pyright`.

## 1) CI Goals

- Prevent spec drift by enforcing lint/type/test gates.
- Catch algorithm correctness and contract regressions early.
- Keep merges deterministic and reviewable.
- Guarantee all Pygame-dependent tests run headlessly without a display server.

## 2) Minimum CI Gates (Locked)

Each PR must pass:

- Environment sync/install via `uv`.
- Lint: `uv run ruff check .`.
- Typecheck: `uv run pyright`.
- Tests: `SDL_VIDEODRIVER=dummy SDL_AUDIODRIVER=dummy uv run pytest -q`.

The `SDL_VIDEODRIVER=dummy` and `SDL_AUDIODRIVER=dummy` environment variables are **mandatory** for the test step. Without them, any test that imports Pygame will crash on `ubuntu-latest` (no display server). The root `tests/conftest.py` also sets these variables as a safety net, but the CI environment must not rely on that alone — setting them at the shell level ensures they are active before any Python import occurs.

## 3) Trigger Policy (Locked)

- Run on:
  - pull requests targeting default branch.
  - pushes to default branch.
- Optional: manual workflow dispatch for ad-hoc reruns.

## 4) GitHub Actions Configuration

```yaml
name: ci

on:
  pull_request:
  push:
    branches: [ main ]

jobs:
  quality:
    runs-on: ubuntu-latest
    env:
      SDL_VIDEODRIVER: dummy
      SDL_AUDIODRIVER: dummy
    steps:
      - uses: actions/checkout@v4
      - name: Install uv
        uses: astral-sh/setup-uv@v4
      - name: Setup Python 3.13
        uses: actions/setup-python@v5
        with:
          python-version: '3.13'
      - name: Sync dependencies
        run: uv sync
      - name: Lint
        run: uv run ruff check .
      - name: Typecheck
        run: uv run pyright
      - name: Unit tests (fast feedback)
        run: uv run pytest -m unit -q
      - name: Full test suite
        run: uv run pytest -q
```

Key differences from a naive configuration:

- `SDL_VIDEODRIVER` and `SDL_AUDIODRIVER` are set at the **job level** via `env:`, making them available to every step. This is safer than per-step `env` because Pygame may be imported during `pyright` type checking as well.
- Unit tests run first (`-m unit`) for fast feedback on algorithm correctness before the full integration suite.
- The full suite (`uv run pytest -q`) runs all tests including integration tests that exercise the Controller with mock generators.

## 5) Optional Enhancements

- Add matrix for Python patch versions inside 3.13 if needed.

- Add coverage >= 90% threshold gate after test suite stabilizes.
- Add `-m "not slow"` to the fast feedback step if slow-marked tests are introduced.

## 6) Merge Expectations

- No direct merge on red pipeline.
- Any waived failure must be documented in PR with:
  - reason,
  - risk assessment,
  - follow-up issue.

## 7) CI Exit Criteria

Pipeline is considered healthy when:

- Recent PRs pass all quality gates (lint/type/unit/integration tests).
- No recurring failures in algorithm correctness or generator contract tests.
- Headless Pygame initialization succeeds consistently (no `pygame.error: No available video device` failures).
